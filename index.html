Step 01 · สัปดาห์ที่ 1–2

Hardware Setup

CT Sensor · Selec EM2M Energy Meter · RS485 Wiring · Modbus RTU — ทำตามได้ทีละขั้นตอน

🔧 อุปกรณ์ที่ต้องใช้: CT 100A/5A (split-core) · Selec EM2M-96 · สาย Shielded twisted pair (RS485) · Termination resistor 120Ω × 2 · USB-RS485 dongle (สำหรับทดสอบจาก PC)
1.1 หลักการวัดไฟฟ้าและ CT Sensor ▾
  • CT (Current Transformer) ทำงานโดยอาศัยการเหนี่ยวนำแม่เหล็ก วัดกระแส AC โดยไม่ต้องตัดวงจร
  • เลือก CT ratio ให้เหมาะกับ load — ถ้า max current 80A ใช้ CT 100A/5A (Safety margin 25%)
  • Split-core CT เปิดคล้องสายได้โดยไม่ต้องตัดไฟ เหมาะสำหรับ retrofit
  • สาย CT ต้องต่อ Burden resistor 5Ω ที่ขั้ว Secondary ของมิเตอร์ (Selec EM2M มีภายในแล้ว)
  • ความปลอดภัย: อย่าเปิดวงจร Secondary CT ขณะมีกระแสไหล — แรงดันจะสูงมาก
1.2 ติดตั้งและตั้งค่า Selec EM2M-96 ▾
  • ติดตั้งบน Din-Rail ขนาด 96×96mm front panel
  • ต่อไฟเลี้ยง: L-N 85–300VAC เข้าขั้ว Power Supply ของมิเตอร์
  • ต่อ Voltage sensing: L1/L2/L3 และ N เข้าขั้ว Voltage input
  • ต่อสาย CT Secondary (5A) เข้าขั้ว Current input I1/I2/I3
  • ตั้งค่าผ่าน front panel: CT ratio → Communication → Modbus Address (1–247) → Baud rate (9600/19200)
  • ตรวจสอบค่าหน้าจอ: V~220V, I~ตามโหลด, PF~0.8–1.0, Hz~50
1.3 RS485 Bus Topology และ Modbus Test ▾
  • RS485 ใช้สาย 3 เส้น: A(+), B(-), GND — ต่อแบบ Daisy-chain ห้ามใช้ Star topology
  • ต่อ Termination resistor 120Ω ที่ปลายทั้งสองด้านของ bus (จาก A+ ถึง B-)
  • ทดสอบด้วย Modbus Poll (PC + USB-RS485): ตั้ง COM port, Baud 9600, 8N1, ใส่ Slave address
  • อ่าน Register 4000 → ต้องได้ค่าแรงดัน (เช่น 2203 = 220.3V หลังหาร 10)
  • Debug: ถ้าไม่ได้ค่า — ตรวจ A/B สลับ, ตรวจ Termination, ตรวจ Address, ตรวจ Baud rate
Step 02 · สัปดาห์ที่ 3–4

PLC Programming

Messung NX-ERA · CODESYS 3.5 · IEC 61131-3 Structured Text · Modbus Master · Alarm Logic

2.1 CODESYS พื้นฐาน — Project Setup ▾
  • Download CODESYS 3.5 SP19+ จาก codesys.com (ฟรี)
  • ติดตั้ง Messung Device Package ผ่าน Tools → Package Manager
  • New Project → Standard Project → เลือก Device: Messung NX-ERA
  • ตั้ง Task: Cyclic, Priority 1, Cycle time 100ms
  • ประเภท Language ที่ควรรู้: ST (Structured Text) เหมือน Pascal · LD (Ladder) สำหรับ Electrician · FBD (Function Block)
2.2 Modbus Master Function Block — Sample Code ▾
STRUCTURED TEXT · IEC 61131-3
(* Poll timer — อ่านทุก 1 วินาที *)
tonPoll(IN := NOT tonPoll.Q, PT := T#1S);

IF tonPoll.Q AND NOT fbModbus.Busy THEN
    fbModbus.Execute  := TRUE;
END_IF

fbModbus(
    Execute    := bExecute,
    SlaveAddr  := 1,        (* Selec Modbus address *)
    FuncCode   := 3,        (* FC03 Read Holding Registers *)
    StartAddr  := 3999,     (* Register 4000 (0-based) *)
    Quantity   := 14,
    DataBuffer := arrRaw,
    Done       => bDone,
    Error      => bError
);

IF bDone THEN
    rVoltage_L1  := WORD_TO_REAL(arrRaw[0]) / 10.0;
    rCurrent_L1  := WORD_TO_REAL(arrRaw[3]) / 10.0;
    rActivePower := WORD_TO_REAL(arrRaw[6]) / 10.0;
    rPowerFactor := INT_TO_REAL(WORD_TO_INT(arrRaw[7])) / 1000.0;
    fbModbus.Execute := FALSE;
END_IF

(* Peak Demand Alarm with Hysteresis *)
IF NOT bAlarm AND rActivePower > 15.0 THEN
    bAlarm := TRUE;
ELSIF bAlarm AND rActivePower < 14.0 THEN
    bAlarm := FALSE;
END_IF
📌 Register address อ้างอิง Selec EM2M Series Manual Rev.2.x — ยืนยันด้วย Modbus Poll ก่อนใช้จริงเสมอ
2.3 Alarm Logic, Peak Demand และ Daily kWh ▾
  • Hysteresis band ป้องกัน output กระพริบ: ON เมื่อ kW > 15 / OFF เมื่อ kW < 14
  • เชื่อม bAlarm → Digital Output %QX0.0 ผ่าน I/O mapping ใน CODESYS
  • Daily kWh reset: ใช้ GET_SYSTEM_TIME() อ่านเวลา PLC → reset เมื่อ Hour = 0
  • Export ข้อมูลผ่าน Modbus TCP หรือ OPC-UA → Node-RED อ่านส่งต่อ MQTT
Step 03 · สัปดาห์ที่ 5–6

IT / Cloud & Dashboard

Node-RED · Mosquitto MQTT · InfluxDB · Grafana — Stack ทั้งหมดรัน Docker บนเครื่องเดียว

3.1 MQTT Protocol — Pub/Sub พื้นฐาน ▾
  • MQTT = Message Queuing Telemetry Transport · โปรโตคอลเบาสำหรับ IoT
  • Publish: อุปกรณ์ส่งข้อมูลไปที่ Broker พร้อม Topic
  • Subscribe: Dashboard รับข้อมูลโดย subscribe topic ที่ต้องการ
  • Topic แนะนำ: factory/building-a/meter01/kW
  • QoS 1 (At least once) เหมาะสำหรับ energy data — ไม่ส่งซ้ำซ้อนเกินไป แต่ไม่สูญหาย
  • ทดสอบด้วย MQTT Explorer (free) ก่อนเชื่อม Grafana
3.2 Node-RED Flow + Docker Stack ▾
  • Node-RED อ่านค่าจาก PLC ผ่าน Modbus TCP node → แปลง JSON → Publish MQTT
  • Docker Compose รัน Mosquitto + InfluxDB + Grafana ในไฟล์เดียว (ดู C2)
  • Telegraf รับ MQTT แล้วเขียนลง InfluxDB อัตโนมัติ ไม่ต้องเขียน code
  • Retention Policy: เก็บ raw 1s นาน 7 วัน · downsampled 15min นาน 1 ปี
3.3 Grafana Dashboard และ Embed ใน Google Sites ▾
  • เชื่อม InfluxDB เป็น Data source: Configuration → Data Sources → InfluxDB
  • สร้าง Panel: Load Profile (Time series), Peak Demand (Stat), PF (Gauge), CO₂ (Stat)
  • ตั้ง Alert Rule: kW > 15 → ส่ง Notification (Email / Line Notify)
  • Embed ใน Google Sites: Share Panel → Embed → Copy URL → Sites → Insert → Embed
  • ใช้ Grafana Cloud Free Tier (3 users, 10k metrics) ถ้าไม่ต้องการ host เอง

D1 Python Anomaly Detection — โค้ดตัวอย่าง ▾
PYTHON 3.10+
from influxdb_client import InfluxDBClient
from sklearn.ensemble import IsolationForest
import pandas as pd

# เชื่อม InfluxDB
client = InfluxDBClient(url="http://localhost:8086",
                          token="YOUR_TOKEN", org="wattiq")
query_api = client.query_api()

# ดึงข้อมูล kW ย้อนหลัง 7 วัน
query = """
from(bucket: "energy")
  |> range(start: -7d)
  |> filter(fn: (r) => r._measurement == "meter" and r._field == "kW")
  |> aggregateWindow(every: 15m, fn: mean)
"""
df = query_api.query_data_frame(query)
df = df[['_time', '_value']].rename(columns={'_value': 'kW'})

# Train Isolation Forest
model = IsolationForest(contamination=0.05, random_state=42)
df['anomaly'] = model.fit_predict(df[['kW']])
anomalies = df[df['anomaly'] == -1]
print(f"พบ {len(anomalies)} จุดผิดปกติ")
print(anomalies[['_time', 'kW']])
